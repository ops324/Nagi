# Nagi ハーネスワークフロー

最終更新: 2026-04-26
バージョン: v1.0.0

## 1. ハーネスの目的

このドキュメントは、Nagi プロジェクトにおける claude.ai と Claude Code の
連携の「足場(harness)」を定義する。

ハーネスを整備する理由:
- 指示書・実装・報告の品質を一定に保つ
- 第7版以前に発生した「動作確認未実施」「ルール漏れ」等の再発防止
- セッションを跨いだ運用ルールの永続化

## 2. 役割分担

| 主体 | 責務 |
|---|---|
| **claude.ai** | 設計・診断・判断・方針決定・仕様書作成・指示書作成 |
| **Claude Code** | 実装・コード調査・動作確認・報告書作成 |
| **滝本さん(人間)** | 指示書承認・Claude Code への投入・報告書レビュー・最終判断 |

## 3. 標準フロー

```
[claude.ai]
  ↓ 1. 課題整理・設計判断
  ↓ 2. Web 検索による仕様確認(22-4)
  ↓ 3. 指示書作成
  ↓ 4. 完成宣言前の再検索(22-6)
  ↓ 5. 滝本さんへ提示
[滝本さん]
  ↓ 6. 指示書レビュー・承認
  ↓ 7. Claude Code へ投入(必要に応じて新セッション)
[Claude Code]
  ↓ 8. バックアップブランチ作成(CC-5)
  ↓ 9. 実装
  ↓ 10. 動作確認(CC-3, CC-4, CC-6)
  ↓ 11. 報告書作成(CC-7)
  ↓ 12. 滝本さんへ提示
[滝本さん]
  ↓ 13. 報告書レビュー
  ↓ 14. claude.ai セッションへ報告書貼り付け
[claude.ai]
  ↓ 15. レビュー・不足点指摘
  ↓ 16. 必要なら追加指示書 or 完了判定
```

## 4. claude.ai 側運用ルール

### 22-1: rules/ 文書の整合性チェックに留める

claude.ai は実装の実在性を判定せず、ドキュメント間の整合性チェックに集中する。
実装の確認は Claude Code に委ねる。

### 22-2: 「実装の実在性」判定は必ず Claude Code 側へ

「このコードは存在するか」「この機能は動くか」の判定は claude.ai では行わない。
必ず Claude Code に確認させる。

### 22-3: 不確実な指摘は「要確認」とマーク

確証がない情報・推測を含む情報は明示的に「要確認」マークを付ける。
断定と推測を混在させない。

### 22-4: フレームワーク仕様は Web 検索で公式確認

フレームワーク・ライブラリの仕様、推奨パターン、最新リリースノートに関わる
判断は、必ず Web 検索で公式ドキュメントを確認してから断定する。

### 22-5: 設計判断前にプロジェクトの将来像を確認

優先度判断・設計判断を行う前に、プロジェクトの将来像
(商用化計画、対象ユーザー、技術スタック方針等)を確認する。

### 22-6: 指示書完成宣言前に設計判断仕様を再度 Web 検索で確認

指示書を「完成」と宣言する前に、設計判断に関わる仕様
(フレームワーク・ライブラリ・公式推奨パターン)を再度 Web 検索で確認する。
22-4 の追加適用ではなく、独立した最終チェックとして実施する。

## 5. Claude Code 側運用ルール

CC-x ルールは `.claude/rules/harness.md` に記載され、
Claude Code 起動時に自動ロードされる。
本ファイルでは概要のみ記載する(詳細は `.claude/rules/harness.md` 参照)。

| ID | 概要 |
|---|---|
| CC-1 | 不確実な指摘・実装は「要確認」とマーク |
| CC-2 | フレームワーク仕様は Web 検索で公式確認 |
| CC-3 | 動作確認チェックリストの全項目を Pass/Fail/Skip 報告 |
| CC-4 | 「ビルド成功」と「動作確認完了」を区別して報告 |
| CC-5 | バックアップブランチ作成有無を必ず報告 |
| CC-6 | console.log 等の確認指示は実ランタイム確認を実施し結果報告 |
| CC-7 | ファイル更新時は差分サマリーを報告(バージョン番号だけは不可) |

## 6. ハーネス自体の改訂ルール

ハーネス(本ファイル + `.claude/rules/harness.md`)に変更が発生した場合:

1. 変更内容を本ファイルの「改訂履歴」セクションに追記
2. バージョン番号を semver で更新
   - メジャー: ルール体系の根本変更
   - マイナー: ルール追加・削除
   - パッチ: 文言修正・誤字訂正
3. `.claude/rules/harness.md` も対応する変更があれば同時に更新
