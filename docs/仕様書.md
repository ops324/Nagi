## 1. プロジェクト概要

### 1.1 アプリ名
**Nagi（凪）** — 自己観察の記録

### 1.2 コンセプト
日記・ジャーナル形式で今の気持ちや思考を記録すると、AIが評価・判断をせず、ユーザー自身が自己観察を深めるためのコメントを返すWebアプリ。感情の抽出・可視化によって、自己理解を静かに深めていく体験を提供する。

### 1.3 デザイン哲学
**Calm Technology × Soft Minimalism**

> 「生産性ツールではなく、静かな日記帳のように感じられるインターフェース」

- **Calm Technology** — 主張せず、そっと寄り添うUI
- **Human-Centered Design** — ユーザーの内省を邪魔しない
- **Soft Minimalism** — 余白と柔らかさで安心感を生む

### 1.4 ターゲットユーザー
- 自己理解を深めたい人
- 感情や思考を整理したい人
- 日記・ジャーナリングを習慣にしたい人

### 1.5 AIキャラクター設定（凪）
| 項目 | 内容 |
|------|------|
| 名前 | 凪（Nagi） |
| 役割 | 自己観察の静かな伴走者 |
| トーン | 温かく、淡々とした語り口 |
| 方針 | 評価・判断をしない。問いかけや観察を通じて気づきを促す |
| コメント文字数 | 100〜150文字程度 |

## 6. データ構造

### 6.1 Entry（記録）型

```typescript
type Emotion = { label: string; score: number; };

type Entry = {
  id: string;          // Date.now()で生成
  user_id: string;     // Supabase auth.users.id（v1.17追加）
  content: string;
  comment: string;
  emotions: Emotion[];
  dominant: string;
  energy: number;      // 1〜10
  createdAt: string;   // ISO 8601
};
```

### 6.2 DBスキーマ（Supabase PostgreSQL）

```sql
-- ユーザープロフィール
public.profiles (
  id uuid PRIMARY KEY,  -- auth.users.id と同一
  email text,
  is_admin boolean DEFAULT false,
  created_at timestamptz
)

-- 日記記録
public.entries (
  id text PRIMARY KEY,
  user_id uuid REFERENCES profiles(id),
  content text,
  comment text,
  emotions jsonb,
  dominant text,
  energy integer,
  created_at timestamptz
)

-- 管理者用ビュー
public.admin_analytics     -- ユーザー別集計（総記録数・最終記録・平均エネルギー）
public.admin_emotion_stats -- 感情ラベル別集計
```

### 6.3 RLS（Row Level Security）ポリシー

| テーブル | 操作 | ポリシー |
|---------|------|---------|
| profiles | SELECT / UPDATE | 自分のレコードのみ（`auth.uid() = id`） |
| entries | SELECT / INSERT / DELETE | 自分のレコードのみ（`auth.uid() = user_id`） |

## 7. 認証フロー（v1.17）

```
未ログインユーザー
    │ アクセス
    ▼
proxy.ts（全ルートをガード）
    │ 未認証 → /auth/login にリダイレクト
    │ 認証済みで /auth/* → / にリダイレクト
    ▼
/auth/login または /auth/signup
    │ Supabase Auth（signInWithPassword / signUp）
    ▼
/ メインページ
    │ Supabase DBからentriesを取得（自分のデータのみ）
    │ 初回ログイン時：SEEDデータを自動挿入
    ▼
ヘッダー
    └── is_admin=true → /admin リンク表示
    └── ログアウトボタン → /auth/login へ
```

### 管理者権限の付与方法

Supabase SQL Editorで実行：
```sql
UPDATE public.profiles SET is_admin = true WHERE email = '管理者のメールアドレス';
```

### 管理者アナリティクスのRLS設定

管理者が全ユーザーデータを参照するために、以下のSQL（`docs/supabase-setup.sql` に含む）を実行が必要：

```sql
-- 管理者権限チェック関数（SECURITY DEFINER でRLSをバイパス）
CREATE OR REPLACE FUNCTION public.is_admin()
RETURNS boolean AS $$
  SELECT COALESCE(
    (SELECT is_admin FROM public.profiles WHERE id = auth.uid()),
    false
  )
$$ LANGUAGE sql SECURITY DEFINER;

-- 管理者は全プロフィール・全エントリを参照可
CREATE POLICY "管理者は全プロフィール参照可" ON public.profiles FOR SELECT
  USING (auth.uid() = id OR public.is_admin());

CREATE POLICY "管理者は全エントリ参照可" ON public.entries FOR SELECT
  USING (auth.uid() = user_id OR public.is_admin());
```

## 11. セキュリティ

| 項目 | 対応 |
|------|------|
| APIキー漏洩防止 | `.env.local` は `.gitignore` で Git 管理外 |
| 認証ガード | `proxy.ts` で全ルートを保護。未ログインは `/auth/login` へリダイレクト |

## 13. AI設計哲学・コンセプトソース

### 13.1 量子力学的視点による自己観測設計
**ソース:** YouTube「【野村泰紀】量子力学の正体」

| 視点 | 概念 | アプリへの応用 |
|------|------|--------------|
| 粒（粒子） | 瞬間のデータは1つの事実 | 素早く記録できるUI |
| 波（分布） | 繰り返しで統計的パターンが現れる | 感情・エネルギーの推移可視化 |

> 自分の人生を「確率の波」として捉え、一時的な落ち込みを「統計的なゆらぎ」として客観視できる設計。

### 13.2 ジャーナリングとAI壁打ちの設計原則
**ソース:** YouTube「テレ東AIアカデミー（田中渓 × 勝間和代）」

- 日記を「言語化能力を鍛える訓練」として設計
- 送信ごとに凪がコメント返信（評価ではなく問いかけ）

### 13.3 自己律・成長のための設計視点
**ソース:** YouTube「イチロー氏 マネジメントで大切にしたいこと」

> 「優しく励ますだけのアプリ」ではなく、ユーザーが自分自身に対して「愛のある戦略的指揮官」になれるツール。

### 13.4 AIコメント設計原則（凪の語り口）

| 原則 | 内容 |
|------|------|
| 非評価 | 「良い／悪い」の判断を下さない |
| 観察促進 | 気づきを深めるための問いかけ |
| 波の視点 | 感情の揺れを統計的ゆらぎとして捉える |
| 重ね合わせの肯定 | 複数感情の同居をそのまま肯定 |
| 物語の連続性 | 単発の記録を自己の物語の一点として位置づける |
| 文字数 | 100〜150文字（簡潔で余白がある） |

### 13.5 「フィルムを変える」視点（多層的な自己観察）
**ソース:** YouTube「【フィルムを変えよう】前提を変えれば人生は変わっていく」

人生を「映画」として捉える3つの視点をAIコメントに織り交ぜる：

| 視点 | 役割 | AIコメントへの応用 |
|------|------|-----------------|
| 演者（スクリーン） | 今この瞬間の感情・体験を生きている自分 | 感情の受容・共感 |
| 観客（客席） | 自分の人生の物語を俯瞰して眺める自分 | 「外から眺める視点」への問いかけ |
| プロデューサー（映写室） | どんな前提・フィルターで世界を見るかを選び直す自分 | パターンへの気づき・新たな選択の可能性を示す |

感情に没入しているときは「外から眺める問い」を届け、前提やパターンに気づいているときは新たな選択の可能性を静かに示す。押しつけず、問いかけの形で自然に織り交ぜる。
